# Config Reference

## Configuration directory

Atlas stores all state in a single directory. The default location is `~/.alma/`.

```
~/.alma/
├── sources.json    registered source connections
└── atlas.db        SQLite asset database (created on first scan)
```

Override the directory with the `ALMA_CONFIG_DIR` environment variable:

```bash
export ALMA_CONFIG_DIR=/etc/alma
alma-atlas scan
```

---

## sources.json

`~/.alma/sources.json` is a JSON array of source objects. Each object has three fields:

| Field | Type | Description |
|-------|------|-------------|
| `id` | string | Unique identifier for this source. Assigned automatically by `alma-atlas connect`. |
| `kind` | string | Adapter type: `bigquery`, `postgres`, or `dbt`. |
| `params` | object | Connection parameters. Shape depends on `kind` — see below. |

The file is managed by `alma-atlas connect` commands. You can also edit it directly.

### BigQuery source

```json
{
  "id": "bigquery:my-gcp-project",
  "kind": "bigquery",
  "params": {
    "project": "my-gcp-project"
  }
}
```

With an explicit service account key file:

```json
{
  "id": "bigquery:my-gcp-project",
  "kind": "bigquery",
  "params": {
    "project": "my-gcp-project",
    "credentials": "/home/user/.config/gcloud/atlas-sa.json"
  }
}
```

| Param | Required | Description |
|-------|----------|-------------|
| `project` | Yes | GCP project ID |
| `credentials` | No | Absolute path to a service account JSON key file. If omitted, Application Default Credentials are used. |

### PostgreSQL source

```json
{
  "id": "postgres:mydb",
  "kind": "postgres",
  "params": {
    "dsn": "postgresql://atlas_user:password@localhost:5432/mydb",
    "schema": "public"
  }
}
```

| Param | Required | Description |
|-------|----------|-------------|
| `dsn` | Yes | PostgreSQL connection string (DSN). Supports all standard `postgresql://` parameters. |
| `schema` | No | Schema to scan. Defaults to `public`. |

The source `id` is derived from the database name in the DSN (`postgres:{dbname}`).

### dbt source

```json
{
  "id": "dbt:my-project",
  "kind": "dbt",
  "params": {
    "manifest_path": "/home/user/my-dbt-project/target/manifest.json"
  }
}
```

With an optional project name override:

```json
{
  "id": "dbt:analytics",
  "kind": "dbt",
  "params": {
    "manifest_path": "/home/user/my-dbt-project/target/manifest.json",
    "project_name": "analytics"
  }
}
```

| Param | Required | Description |
|-------|----------|-------------|
| `manifest_path` | Yes | Absolute path to `manifest.json`. Generated by `dbt compile` or `dbt run`. |
| `project_name` | No | Display name override. Defaults to the project name in the manifest. |

### Multiple sources

```json
[
  {
    "id": "bigquery:my-gcp-project",
    "kind": "bigquery",
    "params": {
      "project": "my-gcp-project"
    }
  },
  {
    "id": "postgres:analytics",
    "kind": "postgres",
    "params": {
      "dsn": "postgresql://atlas_user:secret@db.internal:5432/analytics",
      "schema": "public"
    }
  },
  {
    "id": "dbt:my-project",
    "kind": "dbt",
    "params": {
      "manifest_path": "/home/user/my-dbt-project/target/manifest.json"
    }
  }
]
```

## Environment variables

| Variable | Description |
|----------|-------------|
| `ALMA_CONFIG_DIR` | Override the default config directory (`~/.alma`). Useful for CI, containers, or multi-tenant setups. |

---

## CLI commands for managing sources

```bash
# Register a source
alma-atlas connect bigquery --project my-gcp-project
alma-atlas connect postgres --dsn "postgresql://user:pass@host/db"
alma-atlas connect dbt --project-dir ./my-dbt-project

# List registered sources
alma-atlas connect list

# Remove a source
alma-atlas connect remove bigquery:my-gcp-project
```

## atlas.db

The SQLite database at `~/.alma/atlas.db` (or `$ALMA_CONFIG_DIR/atlas.db`) is created and managed automatically by Atlas. It stores:

- Assets (tables, views, models, seeds, snapshots)
- Edges (lineage relationships between assets)
- Schema snapshots (column names, types, nullability per asset)
- Query fingerprints (deduplicated query traffic)

The database can be safely deleted — it will be rebuilt on the next `alma-atlas scan`. It is not intended for direct querying.
