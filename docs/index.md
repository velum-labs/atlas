# Alma Atlas Docs

Alma Atlas scans warehouses, dbt projects, orchestration systems, and BI layers
into a local graph, then exposes that context through CLI commands and MCP
tools. This site keeps the existing task-focused guides and adds autogenerated
Python API reference from the workspace source tree.

## Workspace Packages

The Python workspace declared in `pyproject.toml` is organized into focused
packages:

| Package | Purpose |
| --- | --- |
| `alma-atlas` | CLI, MCP server, orchestration, learning, sync, and config |
| `alma-atlas-store` | SQLite repositories, migrations, and connection management |
| `alma-connectors` | Source adapters, runtime adapter construction, and connector protocols |
| `alma-analysis` | Pure lineage and graph analysis functions |
| `alma-sqlkit` | SQL parsing, normalization, and extraction helpers |
| `alma-algebrakit` | SQL-agnostic relational algebra and proof utilities |
| `alma-ports` | Shared domain protocols and persisted record types |

## How The API Docs Work

The `Reference > API Reference` section is built with `mkdocstrings` during each
docs build. Package and module pages are generated automatically from public
Python modules, and the page content comes from package/module/class/function
docstrings in the source tree.
