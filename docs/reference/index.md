# Reference

The Atlas reference section combines stable hand-written reference material with
autogenerated Python API documentation.

## Hand-Written Reference

- [Config Reference](../config-reference.md) covers `atlas.yml`, connection
  storage, and runtime behavior.
- [MCP Tools Reference](../mcp-tools.md) documents the Atlas MCP tool surface
  exposed by `alma-atlas serve`.

## Generated API Reference

The API reference is built from public Python modules in each workspace package.
Pages are regenerated during every docs build, so changes to package/module
docstrings show up automatically in the published site.

The generated reference currently covers:

- [`alma_atlas`](api/alma_atlas/index.md)
- [`alma_atlas_store`](api/alma_atlas_store/index.md)
- [`alma_connectors`](api/alma_connectors/index.md)
- [`alma_analysis`](api/alma_analysis/index.md)
- [`alma_sqlkit`](api/alma_sqlkit/index.md)
- [`alma_algebrakit`](api/alma_algebrakit/index.md)
- [`alma_ports`](api/alma_ports/index.md)

## Conventions

- Only public modules are included in the generated API navigation.
- Package, module, class, and function docstrings should follow PEP 257.
- Multi-line API docstrings use Google-style sections such as `Args:`,
  `Returns:`, and `Raises:`.
